# Intent Debt

## 1. 定义

`Intent Debt` 不是删除任务，也不是假装完成。

它表示：

- 当前任务未完成
- 当前路径无法继续
- 但任务意图仍然存在

## 2. 触发条件

典型触发条件：

- 超过最大重试次数
- 缺权限
- 缺配置
- 缺关键需求信息
- 风险超过阈值

## 3. 必须记录的信息

进入 intent debt 时至少记录：

- 任务 id
- 当前目标
- 已做过什么
- 为什么失败
- 为什么不能继续
- 下一步最可能修法

## 4. 当前结论

**Intent Debt 是 Loop 对“没做完但不能乱跑”的正式记账方式。**
