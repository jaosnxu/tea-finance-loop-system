# Sub-Agent System

## 1. 定义

`Sub-Agent` 是 Loop 系统的任务分身机制。

它让系统可以把大任务拆成多个子任务并行推进。

## 2. 什么时候需要子 agent

适合在这些场景启用：

- 多模块开发
- 多来源调研
- 多页面实现
- 多方案并行评估
- 主任务过大，单线程推进过慢

## 3. 子 agent 的基本属性

每个子 agent 应有：

- `id`
- `goal`
- `scope`
- `input_context`
- `assigned_skills`
- `worktree`
- `status`
- `result`

## 4. 子 agent 的边界

子 agent 不能无限放大。

必须限制：

- scope
- 轮数
- 工具权限
- 输出格式

## 5. 子 agent 与主 agent 的关系

主 agent 负责：

- 拆任务
- 分配上下文
- 汇总结果
- 最终判断

子 agent 负责：

- 在限定范围内完成子目标

## 6. 子 agent 输出要求

子 agent 的输出不能只是“我做完了”。

至少要交：

- 做了什么
- 产出了什么
- 有没有问题
- 是否达成子目标

## 7. 子 agent 常见失败点

- 范围太大
- 上下文不足
- 与主目标脱节
- 结果无法汇总

## 8. 子 agent 使用原则

- 能不用就不用
- 必须为复杂任务服务
- 必须有汇总机制
- 必须写回记忆
