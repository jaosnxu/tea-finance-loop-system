# Loop 系统结构定义

## 1. 定义

`Loop` 不是单纯的“反复执行”。

`Loop` 是一个可复用的系统装配方式：

- 五块构件
- 一根记忆脊柱

完整结构为：

1. 自动路线
2. Worktree
3. Skill
4. 连接器（MCP）
5. 子 agent
6. 记忆

## 2. 系统目标

这套系统不是为某一个项目临时服务。

它的目标是：

- 面向不同项目复用
- 让任务可以自动推进
- 让复杂任务可以拆解协作
- 让经验可以沉淀成 skill
- 让外部工具可以标准接入
- 让系统有连续记忆，而不是每次重新开始

## 3. 五块构件

### 3.1 自动路线

自动路线是系统的主执行线。

它负责：

- 接收目标
- 判断当前阶段
- 选择下一步动作
- 执行动作
- 获取反馈
- 决定继续、切换、回退或结束

它解决的是：

- 任务怎么往前跑
- 什么时候停
- 失败后下一轮怎么办

一句话：

**自动路线负责驱动系统持续朝目标前进。**

### 3.2 Worktree

Worktree 是独立工作区。

它负责：

- 给不同任务分配隔离空间
- 给不同尝试分配独立上下文
- 避免多个任务互相污染
- 允许并行尝试不同方案

它解决的是：

- 文件污染
- 项目状态混乱
- 多轮尝试互相覆盖

一句话：

**Worktree 负责把执行现场隔离开。**

### 3.3 Skill

Skill 是系统级可复用能力。

它不是一次性提示词，而是沉淀后的执行规则。

Skill 负责：

- 封装某一类任务的方法
- 规定输入、输出和步骤
- 把经验从“临时发挥”变成“稳定复用”

典型 skill 包括：

- PRD 解析
- 架构拆解
- 数据建模
- 前端页面实现
- 中后台开发
- 调试修复
- 审批流建模

一句话：

**Skill 负责把做事方法沉淀成系统能力。**

### 3.4 连接器（MCP）

连接器是系统与外部世界的接口层。

它负责接入：

- 浏览器
- 文件系统
- API
- 数据库
- 文档系统
- 沟通工具
- 代码平台

它解决的是：

- 系统如何获得外部能力
- 系统如何操作真实环境

一句话：

**连接器负责给系统装上手和眼。**

### 3.5 子 agent

子 agent 是任务分身机制。

它负责：

- 把复杂任务拆成多个子任务
- 让不同 agent 分别处理不同部分
- 把结果汇总回主路线

适合的场景：

- 大型调研
- 多模块代码开发
- 多页面实现
- 多方案并行比较

一句话：

**子 agent 负责把复杂任务拆开并行推进。**

## 4. 一根记忆脊柱

记忆不是一个附属模块，而是贯穿整个系统的主干。

它负责连接：

- 目标历史
- 当前状态
- 已执行动作
- 失败记录
- 成功经验
- Skill 调用记录
- 外部上下文

记忆至少分四层：

1. `任务记忆`
   当前任务做过什么，还剩什么

2. `项目记忆`
   某个项目的结构、规则、历史决策

3. `系统记忆`
   通用运行规则、已有 skill、可调用能力

4. `经验记忆`
   哪些方法常成功，哪些方法常失败

一句话：

**记忆脊柱负责让系统连续，而不是每轮失忆。**

## 5. 六部分如何协作

系统协作顺序如下：

1. 用户给目标
2. 自动路线接管任务
3. 自动路线读取记忆脊柱
4. 判断是否需要调用已有 skill
5. 判断是否需要连接外部工具
6. 判断是否需要新开 worktree
7. 判断是否需要拆出子 agent
8. 执行后把结果写回记忆
9. 再进入下一轮

可压缩成一句：

`目标 -> 自动路线 -> 调用 skill / worktree / MCP / 子 agent -> 写回记忆 -> 下一轮`

## 6. 这套系统和项目的关系

项目不是系统本体。

项目只是输入对象。

系统本体是：

- Loop runtime
- Skill library
- Worktree mechanism
- MCP connector layer
- Sub-agent orchestration
- Memory spine

项目进入系统后，只提供：

- 目标
- 规则
- 文件
- 环境
- 约束

然后由 Loop 系统决定：

- 用哪些 skill
- 开几个 worktree
- 接哪些连接器
- 要不要拆子 agent
- 记忆怎么写入和复用

## 7. 为什么我前面理解偏了

前面的偏差是把 `Loop` 理解成了“循环执行机制”。

这只覆盖了自动路线的一部分。

但你这里的 `Loop` 是一个更完整的系统：

- 不只是会循环
- 还要会隔离
- 会复用 skill
- 会接外部能力
- 会拆 agent
- 会持续记忆

所以正确理解应当是：

**Loop 是系统结构，不只是执行动作。**

## 8. 系统落地时的标准

如果以后要判断一个系统算不算真正的 `Loop`，至少要看这 6 点：

1. 有没有自动路线
2. 有没有独立 worktree
3. 有没有可复用 skill
4. 有没有外部连接器
5. 有没有子 agent 拆分能力
6. 有没有统一记忆脊柱

少掉其中一项，不是不可以运行，但就不是完整的 `Loop` 系统。

## 10. 当前结论

当前应当统一使用这句话来描述：

**Loop = 五块构件 + 一根记忆脊柱。**

其中五块构件是：

- 自动路线
- Worktree
- Skill
- 连接器（MCP）
- 子 agent

记忆作为统一主干，把整套系统串起来。
