# Memory Spine System

## 1. 定义

`Memory Spine` 是 Loop 系统的统一记忆主干。

它不是聊天记录，而是系统连续性的核心。

## 2. 记忆的职责

记忆负责保存：

- 目标
- 状态
- 约束
- 决策
- 已执行动作
- 失败原因
- 成功路径
- 可复用经验

## 3. 四层记忆

### 3.1 任务记忆

记录当前任务的：

- 当前阶段
- 上一轮动作
- 当前阻塞
- 下一步候选

### 3.2 项目记忆

记录某个项目的：

- 技术栈
- 结构约定
- 已知规则
- 历史决策

### 3.3 系统记忆

记录整个 Loop 系统的：

- skill 清单
- connector 清单
- 通用策略
- 默认运行规则

### 3.4 经验记忆

记录跨任务的：

- 哪些方法有效
- 哪些模式常失败
- 哪些 skill 更稳

## 4. 记忆写入原则

- 每轮都写
- 只写关键信息
- 写结构化信息
- 写可复用的信息

## 5. 记忆读取原则

读取记忆不是“全读一遍”。

应按需要取：

- 当前任务优先读任务记忆
- 当前项目优先读项目记忆
- 需要方法时读 skill 和经验记忆

## 6. 记忆失控风险

主要风险：

- 太多无用信息
- 历史冲突
- 旧规则污染当前决策

所以需要：

- 记忆分层
- 记忆摘要
- 冲突标记
- 新旧优先级

## 7. 记忆与 Runtime 的关系

Runtime 每轮都必须：

- 读记忆
- 写记忆

否则系统就只是连续调用，不是完整 loop。
