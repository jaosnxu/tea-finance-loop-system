# Loop System Issue Register

Every Loop-system issue must be recorded here.

Required fields:
- timestamp
- issue_id
- project
- severity
- status
- situation
- symptom
- cause
- impact
- action_taken
- next_step
- related_task_id
- related_pr
- related_check
- regression_test

This is different from `action_log.jsonl`: action log records every runtime step; issue register records defects, blockers, and platform problems that need follow-up.
